# ADR 0003: Complete the TanStack Start migration

Status: accepted

## Context

The website was moved from a Next.js-style architecture to TanStack Start. The migration is now complete, so operational documentation should describe the current system instead of keeping migration-era TODOs and scaffolding notes.

## Decision

Use TanStack Start with TanStack Router file-based routing as the application framework. Vite compiles MDX at build time with `@mdx-js/rollup`, and MDX files import any per-post React components directly.

Fonts are loaded with CSS `@font-face` declarations. Images are plain browser images; photography variants and manifests are generated by the `zipline-sync/` service and served from the local photo mirror.

The production target is Node.js self-hosting behind Nginx on Hetzner/Coolify. The app does not target Cloudflare or Vercel.

## Consequences

- There is no Next.js `localFont` integration.
- There is no `next/image` dependency or runtime image optimizer.
- MDX posts do not use runtime `.components.js` imports.
- Deployment documentation should describe the Node.js self-hosting target, not Cloudflare or Vercel.
